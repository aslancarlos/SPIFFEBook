# Epílogo

Antes de SPIFFE e SPIRE, as organizações enfrentavam um desafio imenso: adotar práticas modernas de infraestrutura causava que seu modelo de segurança antigo se quebrasse. Os perímetros se tornaram cada vez mais porosos e ao mesmo tempo as propriedades de rede como endereços IP se tornaram inúteis para controle de acesso.

SPIFFE e SPIRE ajudam as organizações a enfrentar esse desafio fornecendo identidades únicas e criptograficamente verificáveis para todos os seus serviços. Uma vez que eles têm identidades, os serviços podem implementar funcionalidade de segurança de nível superior, incluindo autenticação, criptografia, observabilidade e autorização.

Fornecer identidades seguras aos serviços primeiro requer uma forma segura de determinar as identidades dos servidores, instâncias de nuvem e clusters em que são executados. Este é o problema da tartaruga inferior que referenciamos no prólogo. SPIFFE e SPIRE resolvem o problema da tartaruga inferior aproveitando múltiplos fatores para fornecer uma prova forte de identidade.

SPIFFE e SPIRE são projetos de código aberto e sempre podem usar mais ajuda! Visite o site em spiffe.io e junte-se ao espaço de trabalho Slack SPIFFE para se envolver com a crescente comunidade de usuários.
